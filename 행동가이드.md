# 무신사-바이마 자동화 행동가이드 (최신)

이 문서는 현재 `auto_shop` 폴더의 실제 코드 기준으로 작성되었습니다.

## 1. 빠른 시작 (Windows)
1. `auto_shop\run.bat` 실행
2. 메뉴에서 필요한 작업 선택

현재 메뉴:
- `1` 필수 자동 설치
- `2` 초기 설정
- `3` 자동화 시작
- `4` 감시 모드
- `5` 바이마 업로드
- `6` 썸네일 편집 (GUI)
- `7` 썸네일 자동 (GUI)
- `8` 링크 이미지 저장
- `9` GUI 실행기 열기
- `0` 종료

참고:
- `run.bat`는 UTF-8 콘솔(`chcp 65001`) 기준으로 동작하도록 맞춰져 있습니다.

## 2. 초기 설정
### 2-1. credentials.json 준비
- Google Cloud에서 서비스 계정 키(JSON) 발급
- 파일명을 `credentials.json`으로 저장
- 위치: `auto_shop\credentials.json`

### 2-2. setup 실행
- `run.bat`에서 `2` 선택 또는 `python setup.py` 실행
- 시트 ID/시트명 입력

## 3. 시트 상태값 기준 (현재 로직)
`main.py` 기준 핵심 상태:
- `신규`
- `보류`
- `정찰중`
- `정찰완료`
- `이미지저장중`
- `이미지저장완료`
- `썸네일작업중`
- `썸네일완료`
- `업로드중`
- `출품완료`
- `오류`

워커 입력 상태(권장 흐름):
- 정찰 워커: `신규` (또는 빈 상태)
- 이미지 워커: `정찰완료`
- 썸네일 워커: `이미지저장완료`
- 업로드 워커: `썸네일완료`

공통 규칙:
- 작업 시작 즉시 실행중 상태로 먼저 변경
- `출품완료` 행은 모든 단계에서 스킵

## 4. 감시 모드 동작
### 4-1. 메인 감시
- 실행: `python main.py --watch`
- 기본 주기: 20초 (`--interval`로 변경 가능)
- 시트를 반복 확인하며 단계별 대상 행을 처리

### 4-2. 업로드 감시 (`buyma_upload.py`)
- 실행: `python buyma_upload.py --watch --mode auto`
- 감시모드는 `--row`를 고정하지 않고 전체 대기 행을 순차 처리
- 한 건 처리 후 브라우저를 닫고 다음 점검으로 진행
- 감시모드에서는 Enter 대기(수동 입력) 없이 비대화식으로 동작

## 5. 업로드 모드
- `--mode auto`: 오류 없으면 자동 제출까지 진행
- `--mode review`: 확인용(수동 검토 중심)

권장:
- 실제 운영은 `auto`
- 신규 셀렉터 점검/폼 변경 확인 시 `review`

## 6. 자주 발생하는 문제
### 6-1. 콘솔 한글 깨짐
- 최신 `run.bat` 사용
- 기존 콘솔 창 완전히 닫고 다시 실행
- 파일 공유는 ZIP 권장 (메신저 미리보기/복붙 공유 지양)

### 6-2. credentials.json 오류
- 파일 위치 확인: `auto_shop\credentials.json`
- 서비스 계정 메일이 시트에 편집 권한인지 확인

### 6-3. 감시모드가 다음 행으로 안 넘어감
- 최신 코드에서는 수정 완료
- 구버전 실행 중이면 pull/덮어쓰기 후 재실행

## 7. 권장 실행 순서
1. `run.bat` → `1` (최초 1회)
2. `run.bat` → `2` (초기 설정)1
3. 시트에 URL/필수값 입력
4. `run.bat` → `4` (감시 모드) 또는 목적별 개별 실행

## 8. 주요 실행 명령어
```powershell
# 메인 1회 실행
python auto_shop/main.py

# 메인 감시모드 (20초)
python auto_shop/main.py --watch --interval 20

# 업로드 1회 자동
python auto_shop/buyma_upload.py --mode auto

# 업로드 감시모드 자동
python auto_shop/buyma_upload.py --watch --mode auto --interval 20
```
