# 무신사→바이마 자동화 - 행동가이드 (공유용)

이 문서는 처음 받는 사람도 그대로 따라 실행할 수 있게 만든 설치/운영 가이드입니다.

소요시간:
- 초기 설정: 약 15~20분 (1회)
- 이후 사용: 1~2분

---

## 1. 준비물

- Google 계정 1개
- 무신사 상품 URL이 들어갈 Google Sheets 1개
- 이 폴더 전체 파일
- 운영체제: Windows 또는 macOS

주의:
- credentials.json은 개인 인증키이므로 절대 외부 공유 금지

---

## 2. Google Cloud 설정 (처음 1회)

1. https://console.cloud.google.com 접속
2. 새 프로젝트 생성
3. Google Sheets API 활성화
4. 서비스 계정 생성 (권한: Editor)
5. 서비스 계정 키를 JSON으로 발급
6. 파일명을 credentials.json으로 맞춘 뒤 auto_shop 폴더에 저장

확인 포인트:
- auto_shop 폴더 안에 credentials.json 존재

---

## 3. Google Sheets 준비

이 프로젝트는 아래 열 구조를 기준으로 동작합니다.

- A: 순서
- B: 링크
- C: 브랜드이름
- D: 한국상품명
- E: 영문상품명 (수식/수동 입력용)
- F: 색상
- G: 영문색상 (수식/수동 입력용)
- H: 사이즈
- I: 상품가격
- J: 바이마 판매가격 (품명 검색 기반 최저가 자동 입력)
- K: 해외배송료
- L: 등록여부

입력 규칙:
- 사용자는 B열(링크)에 무신사 URL만 넣으면 됨
- 기존에 값이 있는 셀은 덮어쓰지 않음
- J열은 비어 있을 때만 자동 입력됨 (검색 실패 시 공란 유지)

공유 권한:
1. 스프레드시트 우측 상단 공유 클릭
2. credentials.json 안의 client_email 추가
3. 권한은 편집자 부여

## 4. 자동 설치 (Python 포함)

### Windows

1. run.bat 실행
2. 메뉴에서 0번 필수 자동 설치 선택
3. 프로그램처럼 쓰려면 메뉴 5번 GUI 실행기 열기 선택
4. 아이콘 달린 앱 파일이 필요하면 메뉴 6번 아이콘 앱 빌드 선택

자동으로 수행되는 작업:
- Python 미설치 시 winget으로 Python 설치 시도
- .venv 가상환경 생성
- requirements.txt 패키지 설치

### macOS

1. run.command 실행
2. 메뉴에서 0번 필수 자동 설치 선택
3. 프로그램처럼 쓰려면 메뉴 5번 GUI 실행기 열기 선택
4. 아이콘 달린 앱 파일이 필요하면 메뉴 6번 아이콘 앱 빌드 선택

자동으로 수행되는 작업:
- Python 미설치 시 Homebrew로 Python 설치 시도
- .venv 가상환경 생성
- requirements.txt 패키지 설치

참고:
- macOS에서 Homebrew가 없으면 설치 안내 문구가 출력됨
- 빌드 결과 파일: dist/AutoShopLauncher.app(macOS), dist/AutoShopLauncher.exe(Windows)
- macOS 아이콘: images/app_icon.icns가 있으면 적용, 없으면 시스템 아이콘 사용
- Windows 아이콘: images/app_icon.ico가 있으면 적용

## 5. 초기 연결 설정

1. run.bat 또는 run.command 실행
2. 메뉴에서 1번 설정 선택
3. 안내에 따라 Google Sheets ID 입력

Sheets ID 찾는 방법:
- 스프레드시트 주소
- https://docs.google.com/spreadsheets/d/여기가ID/edit

---

## 6. 실제 실행 방법

일회 실행:
- 메뉴 2번 자동화 시작

감시 모드(새 링크 자동 반영):
- 메뉴 3번 감시 모드 시작

중지 방법:
- 터미널 창 X 버튼으로 닫기
- 또는 Ctrl + C

---

## 7. 팀 공유용 운영 규칙

1. 폴더 전체를 공유할 때는 credentials.json 제외
2. 각 사용자 PC에서 본인 credentials.json만 따로 넣기
3. run.bat 또는 run.command의 0번부터 순서대로 실행
4. 시트 열 구조(A~L)는 임의로 바꾸지 않기
5. 링크는 B열만 사용하기

### 같은 Google Sheets를 2명이 같이 쓸 때

- 시트 파일은 따로 복사하지 않고 같은 시트를 함께 사용하면 됨
- 자동화를 둘 다 실행할 예정이면 각자 Google Cloud에서 본인 credentials.json을 발급받아 각자 PC에만 저장
- 시트 공유에는 각 credentials.json 안의 client_email을 모두 편집자로 추가
- 자동화를 한 사람만 실행할 예정이면 그 사람 PC에만 credentials.json이 있으면 됨
- credentials.json 파일은 서로 주고받지 않는 것을 권장

## 8. 자주 발생하는 문제

credentials.json not found:
- auto_shop 폴더에 credentials.json이 있는지 확인

401 Unauthorized:
- 스프레드시트가 서비스 계정 이메일과 공유되어 있는지 확인
- 권한이 편집자인지 확인

Python 설치 실패 (Windows):
- winget 사용 가능 여부 확인
- winget이 없으면 python.org에서 Python 설치 후 재실행

Python 설치 실패 (macOS):
- Homebrew 설치 후 재실행

상품 정보 일부 누락:
- 해당 URL만 다시 실행
- 계속 재현되면 URL 목록을 따로 모아 점검

- 2번 또는 3번 실행
